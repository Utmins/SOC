* General *
***********

Autopsy — это ведущая платформа криминалистики с открытым исходным кодом, которая является быстрой, простой в использовании и способна анализировать все типы мобильных устройств и цифровых носителей.
Ее архитектура подключаемых модулей обеспечивает расширяемость с помощью разработанных сообществом или специально созданных модулей.
Autopsy создана, чтобы удовлетворить потребности специалистов в области правоохранительных органов, национальной безопасности, поддержки судебных разбирательств и корпоративных расследований

Интерент ресурсы    -	https://www.autopsy.com/

                        https://sleuthkit.org/autopsy/docs/user-docs/4.12.0/

* Workflow *
************

Прежде чем начать работу в Autopsy, необходимо выполнить несколько шагов, например, определить источник данных и действия Autopsy, которые следует выполнить с источником данных.

Базовый рабочий процесс:

    -   Создать/открыть дело для источника данных, который вы будете исследовать
    -   Выбрать источник данных, который вы хотите проанализировать
    -   Настроить модули ingest для извлечения определенных артефактов из источника данных
    -   Просмотреть артефакты, извлеченные модулями ingest
    -   Создать отчет

    +++++++++++++++++++++++++++++++++++++
    + Case Analysis | Create a New Case +
    +++++++++++++++++++++++++++++++++++++

    Чтобы подготовить новое расследование дела, вам необходимо создать файл дела из источника данных.
    При запуске Autopsy будет три варианта
    
        1)  New Case

            Вы можете создать новый файл дела с помощью опции «New Case».
            После нажатия на опцию «New Case» откроется меню «Case Information», в котором будет указана информация о деле.

                -   Case Name
                    Имя, которое вы хотите дать делу
                -   Base Directory
                    Корневой каталог, в котором будут храниться все файлы, относящиеся к делу (будет отображен полный путь)
                -   Case Type
                    Укажите, будет ли это дело локальным (однопользовательским) или размещенным на сервере, где его смогут просматривать несколько аналитиков (многопользовательским)

            Следующий экран называется «Optional Information» и может быть оставлен пустым для ваших целей.
            В реальной судебно-медицинской среде вы должны заполнить эту информацию.
            После нажатия кнопки «Finish» Autopsy создаст новый файл дела на основе указанного источника данных.

        2)  Open Recent Case

        3)  Open Case

            Autopsy также может открывать готовые файлы дел.
            *** Примечание: файлы дел Autopsy имеют расширение файла «.aut».

            Чтобы открыть дело, выберите опцию «Открыть дело».
            Перейдите в папку дела (расположенную на рабочем столе) и выберите файл .aut, который вы хотите открыть.
            Затем Autopsy обработает файлы дел и откроет дело.
            Вы можете определить имя дела в левом верхнем углу окна Autopsy.

            !!! Примечание: Если Autopsy не сможет найти образ диска, появится окно с предупреждением.
                На этом этапе вы можете указать местоположение образа диска, который он пытается найти, или нажать НЕТ; вы по-прежнему можете анализировать данные из дела Autopsy.
    
    ++++++++++++++++
    + Data Sources +
    ++++++++++++++++

    Autopsy может анализировать несколько форматов образов дисков.
    Прежде чем погрузиться в этап анализа данных, давайте кратко рассмотрим различные источники данных, которые может анализировать Autopsy.
    Вы можете добавить источники данных, используя кнопку «Add Data Source».
    *** В используемых нами примераъ мы сосредоточимся в первую очередь на параметрах «Disk Image» или «VM File option».

    Поддерживаемые форматы образов дисков:

        -   Raw Single (например, *.img, *.dd, *.raw, *.bin)
        -   Raw Split (например, *.001, *.002, *.aa, *.ab и т. д.)
        -   EnCase (например, *.e01, *.e02 и т. д.)
        -   Виртуальные машины (например, *.vmdk, *.vhd)
        *** Если есть несколько файлов образов (например, E01, E02, E03 и т. д.), Autopsy нужно указать только на первый файл образа, а Autopsy обработает все остальное.  
            Расширение файла E01 обозначает формат файла изображения EnCase, используемый программным обеспечением EnCase.

    Если хотите узнать  о других источниках данных, которые можно добавить в дело, то смотрите документацию тут - https://sleuthkit.org/autopsy/docs/user-docs/4.12.0/ds_page.html

    ++++++++++++++++++
    + Ingest Modules +
    ++++++++++++++++++

    По сути, модули Ingest являются подключаемыми модулями Autopsy.
    Каждый модуль Ingest предназначен для анализа и извлечения определенных данных с диска.
    Вы можете настроить Autopsy для запуска определенных модулей на этапе добавления источника или позже, выбрав целевой источник данных, доступный на панели управления.
    По умолчанию модули Ingest настроены для работы со всеми файлами, каталогами и нераспределенным пространством.
    Вы можете изменить эту настройку на этапе выбора модуля.
    Вы можете отслеживать процесс с помощью панели, появляющейся в правом нижнем углу.

    !!! Обратите внимание, что для реализации использования модулей Ingest требуется время.

    !!! Также Autopsy добавляет метаданные о файлах в локальную базу данных, а не фактическое содержимое файлов.

    Если Вы хотите использовать модули Ingest после добавления источников данных, то:

        -   Откройте меню «Run Ingest Modules», щелкнув правой кнопкой мыши по источнику данных.
        -   Выберите модули для внедрения и нажмите кнопку «finish».
        -   Отслеживайте ход внедрения.
    
    Обратите внимание, что результаты зависят от набора данных.
    Результаты любого модуля Ingest, выбранного для запуска с источником данных, будут заполнять узел Results в представлении Tree, которое является левой панелью пользовательского интерфейса Autopsy.
    Если вы выберете модуль для извлечения определенных данных, которые недоступны на диске, результатов не будет.

    Возвращаясь к окну Configure Ingest Modules, обратите внимание, что некоторые Ingest Modules имеют настройки для каждого запуска, а некоторые — нет.
    Например, у Keyword Search Ingest Module нет настроек для каждого запуска.
    Напротив, у Interesting Files Finder Ingest Module они есть.
    Желтый треугольник представляет собой «параметр настроек для каждого запуска».

    Во время работы Ingest Modules в Ingest Inbox могут появляться оповещения.

    Чтобы узнать больше о модулях Ingest, ознакомьтесь с документацией Autopsy - https://sleuthkit.org/autopsy/docs/user-docs/4.12.0/ingest_page.html

    ++++++++++++++++++++++
    + The User Interface +
    ++++++++++++++++++++++

    Давайте рассмотрим пользовательский интерфейс Autopsy, который состоит из 5 основных областей:

        ///////////////////
        /// Tree Viewer ///
        ///////////////////

        В Tree Viewer есть пять узлов верхнего уровня:

            -   Data Sources
                Все данные будут организованы так, как вы обычно видите их в обычном проводнике Windows.
            -   Views
                Файлы будут организованы на основе типов файлов, типов MIME, размера файлов и т. д.
            -   Results
                Здесь будут отображаться результаты из Ingest Modules.
            -   Tags
                Будут отображать файлы и/или результаты, которые были помечены тегами (подробнее о тегировании читайте здесь).
            -   Reports
                Будут отображать отчеты, созданные модулями или аналитиком (подробнее об отчетах читайте здесь).
        
        *** Более подробную информацию см. в документации Autopsy по Tree Viewer.

        /////////////////////
        /// Result Viewer ///
        /////////////////////

        !!! Примечание: не путайте узел Results (из Tree Viewer) с Result Viewer.

        Когда в Tree Viewer выбран том, файл, папка и т. д., в Result Viewer отображается дополнительная информация о выбранном элементе.
        Например, выбран источник данных Sample case, и теперь дополнительная информация отображается в Results Viewer.

        Обратите внимание, что Result Viewer панель имеет три вкладки:
        
            -   Table
            -   Thumbnail
            -   Summary
        
        Вкладка Thumbnail лучше всего работает с файлами изображений или видео.
        Если вид указанных выше данных изменить с Table на Thumbnail, будет отображаться не так много информации. 

        Узлы тома можно расширять, и аналитик может перемещаться по содержимому тома, как в обычной системе Windows.

        В Views tree node файлы классифицируются по типам файлов
        
            -   By Extension
            -   By MIME Type
            -   Deleted Files
            -   By File Size.
        
        !!! Совет: Когда дело доходит до типов файлов, обратите внимание на этот раздел.
            Злоумышленник может переименовать файл с помощью вводящего в заблуждение расширения файла.
            Таким образом, файл будет «неправильно категоризирован» By Extension, но будет правильно категоризирован By MIME Type.
            Разверните «By Extension», и появятся дополнительные дочерние узлы, еще больше категоризируя файлы

        ///////////////////////
        /// Contents Viewer ///
        ///////////////////////

        Если щелкнуть любую папку/файл на вкладке «Table» в окне Result Viewer, на панели Contents Viewer отобразится дополнительная информация.
        В отображаемом результате Вы можете увидеть 3 столбца (S / C / O) которые могут быть не сразу поняты, что они представляют.

            -   S = Score
                В оценке будет отображаться красный восклицательный знак для папки/файла, помеченного/тегированного как примечательный,
                И желтый треугольник, направленный вниз, для папки/файла, помеченного/тегированного как подозрительный.
                Эти элементы могут быть помечены/тегированы модулем Ingest или аналитиком.

            -   C = Comment
                Если в столбце «Comment» отображается желтая страница, это будет означать, что для папки/файла есть комментарий.

            -   O = Occurrence 
                Вкратце, этот столбец будет указывать, сколько раз этот файл/папка были замечены в прошлых случаях (для этого потребуется центральный репозиторий)

        Дополнительную информацию см. в документации по вскрытию в окне просмотра содержимого - https://sleuthkit.org/autopsy/docs/user-docs/4.12.0/content_viewer_page.html

        //////////////////////
        /// Keyword Search ///
        //////////////////////

        В правом верхнем углу вы найдете Keyword Lists и Keyword Search.
        С помощью Keyword Search аналитик может выполнить поиск ключевых слов AD-HOC

        Дополнительную информацию о выполнении поиска по ключевым словам с использованием любого из этих вариантов см. в документации Autopsy - https://sleuthkit.org/autopsy/docs/user-docs/4.12.0/ad_hoc_keyword_search_page.html

        ///////////////////
        /// Status Area ///
        ///////////////////

        Status Area находится внизу справа.
        Когда Ingest Modules запущены, в этой области будет отображаться полоса прогресса (вместе с процентом выполнения).
        Более подробная информация о Ingest Modules будет предоставлена, если вы нажмете на полосу.

        Если нажать X (прямо рядом с индикатором выполнения), появится запрос на подтверждение, хотите ли вы завершить/отменить Ingest Modules.

        Ознакомьтесь с документацией Autopsy по обзору пользовательского интерфейса здесь - https://sleuthkit.org/autopsy/docs/user-docs/4.12.0/uilayout_page.html
    
    Давайте посмотрим, где мы можем легко найти обобщенную информацию.
    Обобщенная информация может помочь аналитикам решить, на чем сосредоточиться, оценив доступные артефакты.
    Рекомендуется просмотреть сводку источников данных перед началом расследования.
    Таким образом, вы можете получить общее представление о системе и артефактах.

        ////////////////////////////
        /// Data Sources Summary ///
        ////////////////////////////

        Data Sources Summary содержит обобщенную информацию по девяти различным категориям.
        Обратите внимание, что это обзор всех результатов.
        Если вы хотите углубиться в результаты и найти определенный артефакт, вам нужно проанализировать каждый модуль отдельно, используя «Result Viewer».

        ///////////////////////
        /// Generate Report ///
        ///////////////////////

        Вы можете создать отчет о своих результатах в нескольких форматах, что позволит вам создавать листы данных для вашего дела о расследовании.
        Отчет содержит всю информацию, указанную на панели «Result Viewer».
        Отчеты могут помочь вам повторно исследовать результаты после завершения живого расследования.
        Однако отчеты не имеют дополнительных параметров поиска, поэтому вам придется вручную находить артефакты для интересующего события.

        !!! Совет: инструмент Autopsy может быть тяжелым для систем с низкими ресурсами.
            Поэтому завершение расследования с помощью Autopsy при низких ресурсах может быть медленным и болезненным.
            Особенно просмотр длинных результатов может закончиться зависанием системы.
            Вы можете избежать этой ситуации, используя отчеты.
            Вы можете использовать инструмент для анализа данных и создания отчета, а затем продолжить анализ с помощью созданного отчета без необходимости в Autopsy.

            Вы можете использовать опцию «Generate Report» для создания отчетов

                1)  Нажимаем на вкладку Generate Report
                2)  Выбираем в каком виде будет создан отчет (html, excel, text, TSK, google earth KML, STIX, CASE-UCO, Portable case)
                3)  Выбираем источник информаци для создания отчета
                4)  Выбираем какие именно раздела отчета должны быть сформированны
                5)  Просмтриваем сформированный отчет

    Инструмент «Timeline tool» состоит из трех областей:

        i)      Filters
                Сужение отображаемых событий на основе критериев фильтра
        ii)     Events
                События отображаются здесь на основе режима просмотра
        iii)    Files/Contents
                В этой области отображается дополнительная информация о событиях

    Существует три режима просмотра:

        i)      Counts
                Количество событий отображается в виде столбчатой ​​диаграммы
        ii)     Details
                Информация о событиях отображается, но они сгруппированы и свернуты, поэтому пользовательский интерфейс не перегружен
        iii)    List
                События отображаются в виде таблицы
